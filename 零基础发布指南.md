# 🚀「每一天」零基础发布指南

> 目标：不写一行代码，把 App 打包成 APK 发给朋友安装。
> 预计时间：30-40 分钟（主要是等待时间）
> 需要花钱吗：**完全免费**

---

## 📖 第一部分：什么是 GitHub？为什么用它？

简单理解：
- **GitHub** = 一个存代码的网盘 + 一台免费帮你打包的电脑
- 你把代码传上去 → GitHub 自动帮你生成 APK → 你下载发给别人

全程你只需要在网页上点点按钮，**不需要在你自己电脑上装任何东西**。

---

## 🧾 第二部分：注册 GitHub 账号（5 分钟）

### 第 1 步：打开网站

在浏览器地址栏输入：**github.com**，回车。

### 第 2 步：点注册按钮

页面右上角（或中间）有一个 **Sign up** 按钮，点击它。

> 如果页面是英文的，没关系，跟着我的步骤走就行。

### 第 3 步：输入邮箱

看到一栏让你输入 **email**，输入你的邮箱地址。

> 推荐用 QQ 邮箱（例如 `123456789@qq.com`），因为微信/QQ 能收到通知。

### 第 4 步：设置密码

下一栏 **password**，设置一个密码。

> ⚠️ 密码要求：至少 8 位，包含字母和数字。例如：`meitian2026`

### 第 5 步：设置用户名

下一栏 **username**，取一个英文用户名（全网站唯一）。

> 建议：`meitian` + 你的名字拼音，例如 `meitianxiaoming`。
> 如果提示已被占用，在后面加几个数字即可，如 `meitianxiaoming2026`

### 第 6 步：验证你不是机器人

会弹出一个拼图或图片验证，跟着提示做就行。

### 第 7 步：输入验证码

GitHub 会往你邮箱发一封验证邮件，里面有一个 6 位数字验证码。
- 打开你的邮箱（QQ 邮箱的话去 mail.qq.com）
- 找到 GitHub 发的邮件（可能在垃圾箱里）
- 把验证码输入到网页上

### 第 8 步：完成注册

点 **Create account**。看到欢迎页面说明注册成功！

> 📌 **记好你的邮箱和密码！以后还要用！**

## 📦 第三部分：创建仓库（3 分钟）

### 第 1 步

登录后，点页面左上角的绿色 **New** 按钮，或者点右上角 **+** → **New repository**。

### 第 2 步：填写仓库信息

| 栏位 | 填写内容 | 说明 |
|------|---------|------|
| Repository name | `everyday` | 仓库名字 |
| Description | `每一天 App` | 可选，描述一下 |
| Public / Private | 选 **Public** | 公开的（免费） |

### 第 3 步

其他选项**全部不用管**，直接拉到页面最下面，点绿色的 **Create repository** 按钮。

---

## 📤 第四部分：上传代码（5 分钟）

现在你看到了一个空仓库页面，上面显示了一些英文。我们用它提供的上传功能：

### 第 1 步

页面上找到蓝色的 **uploading an existing file** 链接，点击它。（如果没有看到，往上滚动一点）

### 第 2 步：拖拽上传

你会进入一个文件上传页面。
1. 打开你电脑上的文件夹，找到 `everyday` 文件夹
2. **把整个 `everyday` 文件夹里的所有内容**，全部选中，直接拖到浏览器窗口里

> 你的 `everyday` 文件夹在：`C:\Users\Administrator.DESKTOP-5AR47E9\Desktop\daydayday\everyday\`

### 第 3 步：确认上传

等文件列表加载完（可能有几百个文件，正常），页面下方有两个输入框：

- 第一个输入框（Commit message）：输入 `初始化项目`
- 第二个（Description）：不用填

### 第 4 步

点绿色的 **Commit changes** 按钮。

## 🤖 第五部分：设置自动打包（10 分钟）

### 第 1 步：创建 Actions 配置

现在回到你的仓库主页（网址是 `github.com/你的用户名/everyday`）。

1. 点上方的 **Actions** 标签
2. 看到 "Get started with GitHub Actions" → 点 **set up a workflow yourself** 链接
3. 你会进入一个代码编辑页面

### 第 2 步：粘贴配置

把编辑框里原来的内容**全部删掉**，然后把下面这段代码**完整复制粘贴进去**：

```yaml
name: 自动打包APK

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: 安装 Flutter
        uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.19.6'
          channel: 'stable'

      - name: 安装依赖
        run: |
          flutter pub get
          # 创建临时 Firebase 占位文件（不需要真实配置也能打包）
          echo '{}' > android/app/google-services.json

      - name: 打包 APK
        run: flutter build apk --release --split-per-abi

      - name: 上传 APK 文件
        uses: actions/upload-artifact@v4
        with:
          name: 每一天-APK
          path: build/app/outputs/flutter-apk/*.apk
```

### 第 3 步：保存

点右上角绿色的 **Commit changes...** 按钮 → 弹窗再点 **Commit changes**。

### 第 4 步：等待自动打包

保存后，GitHub 会自动开始打包！
1. 点 **Actions** 标签
2. 你会看到一个正在运行的黄色圆圈（表示打包中）
3. 等大约 5-8 分钟，圆圈变成绿色 ✅ 就说明打包完成了

> 这期间你可以关掉网页，过一会再回来看。

### 第 5 步：下载 APK

打包完成后：
1. 点上方的 **Actions** 标签
2. 点最新的那一条（名字是你刚才写的 commit message）
3. 往下翻，找到 **Artifacts** 区域
4. 点 **每一天-APK** 下载

## 📲 第六部分：分享给朋友

下载的 zip 文件解压后，里面有这些文件：

| 文件名 | 给谁用 |
|--------|--------|
| `app-armeabi-v7a-release.apk` | 大部分安卓手机 |
| `app-arm64-v8a-release.apk` | 较新的安卓手机（推荐发这个） |
| `app-x86_64-release.apk` | 安卓模拟器用的，不用管 |

**推荐把 `app-arm64-v8a-release.apk` 发给朋友。**

### 发给朋友的方式：
- **微信**：直接发文件（可能会提示安全风险，点「继续发送」即可）
- **QQ**：直接发文件
- **网盘**：上传到百度网盘/阿里云盘，分享链接

### 朋友怎么安装：
1. 下载 APK 文件
2. 点开文件 → 提示「禁止安装未知来源应用」→ 点「设置」→ 开启「允许来自此来源」
3. 返回，点「安装」
4. 安装完成，桌面出现「每一天」图标 🎉

## 🔄 以后怎么更新？

当你修改了代码想重新打包：
1. 把改好的文件拖到 GitHub 仓库（和第四部分一样）
2. GitHub **自动**帮你重新打包
3. 等几分钟去 Actions 下载新 APK
